VoterMart™ Geoserver
Release Notes

Version 1.1

The VoterMart Geoserver provides geospatial web services including geocoding street addresses and political district coding.  This application is built on the Manifold GIS runtime engine.  The following states are included in the geodatabase:  California.

Version Notes

Version 1.1

  • Results returned as XML formatted type.
  • Provide contact information for both state house legislators.
  • Provide contact information for federal senators.

Version 1.0

  • Provides contact information lookup for California legislators by lower state house, upper state house and congressional district.
  • Supports one or more states legislative tables.
  • Improved error status messaging in results.

Version 0.1

  • Provides GeocodeService for street address geocoding based on the TIGER street database.
  • Provides DistrictCodeService that geocodes a street address and returns lower state house, upper state house, and congressional district numbers.
  • District geodatabase included supports only the state of California in this version.
  • The geodatabase must be in a fixed location for the web services to work.  This will be addressed in a future version.
  • Add GeocodeRawAddressWithStruct web method for geocoding and returning GeoMatch type.

Software Requirements

The VoterMart Geoserver requires the following software in order to operate correctly:

  • Windows Server 2003/2008
  • IIS 6.0 or greater
  • .NET Framework 3.5SP1 (required)
  • Manifold geocoder database
  • Manifold Runtime 8.0 with geocoder module

Known Issues

  • The VoterMart Geoserver will not run correctly with Microsoft .NET Framework 4 due to issues in the Manifold Runtime libraries.  You must install .NET Framework 3.5 SP1.
  • While the Manifold GIS runtime engine supports a variety of different geocoding services including Microsoft MapPoint, there are many issues involving the use of the MapPoint database (which would be more accurate and extensive than TIGER) not to mention licensing issues.
